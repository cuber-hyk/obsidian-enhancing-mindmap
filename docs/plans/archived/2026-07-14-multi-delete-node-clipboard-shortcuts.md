# 多选批量删除与节点剪贴板快捷键计划

## 目标

让多选节点可通过 `Backspace`/`Delete` 一次删除并整体撤销，同时让现有单节点剪切、复制、粘贴及脑图撤销在非编辑态使用标准 `Ctrl/Cmd+X/C/V/Z`，且不干扰文字编辑和 Obsidian 其他视图。

## 范围

- 范围内：多选选择根计算；原子批量删除、撤销和重做；删除后的活动节点回退；单节点剪贴板操作统一入口；脑图焦点内标准剪贴板与撤销快捷键；现有命令面板入口接线；移除旧 `Alt+Shift+C/X/V/Z` 默认热键。
- 范围外：多节点复制、剪切或粘贴；`Ctrl/Cmd+Y` 或 `Ctrl/Cmd+Shift+Z` 重做快捷键；批量删除确认弹窗；新工具栏或右键菜单；剪贴板数据格式升级；跨应用节点格式；节点文字编辑态的原生剪贴板与撤销行为；附件文件删除。

## 计划就绪检查

- 目标明确：是；两项可观察结果分别是多选批量删除和非编辑态标准单节点剪贴板快捷键。
- 范围明确：是；多选仅增加删除，不扩展多节点剪贴板。
- 事实来源已知：是；多选集合由 `NodeSelectionController` 管理，节点结构变更由 History 命令管理，单节点序列化由 `MindMap.copyNode()/pasteNode()` 管理。
- 关键决策已确认：是；用户于 2026-07-14 确认保留多选批量删除，只调整现有单节点剪贴板快捷键。
- 验证路径已知：是；生产构建加测试 Vault 的键盘、树结构和撤销/重做矩阵。

## 假设与决策

- “back” 按键按已确认语义解释为 `Backspace`，同时支持 `Delete`；不新增界面删除按钮。
- 多选删除只处理没有已选祖先的选择根，父节点与后代同时选中时不重复删除后代；选择根顺序沿用脑图树顺序。
- 批量删除通过一条 `RemoveNodes` History 命令执行；一次撤销恢复全部节点的原父节点、索引和稳定顺序，一次重做再次删除全部。
- 批量删除不弹确认框，沿用现有单节点删除交互；删除后以原主选节点为基准，优先选择仍存在的相邻兄弟，否则选择其父节点。
- 节点剪贴板仍只处理唯一活动节点和其完整子树，不改变现有 JSON 数据格式或“粘贴为当前活动节点子节点”的语义。
- `Ctrl/Cmd+C/X/V` 只在脑图画布获得焦点、存在唯一活动节点且节点不处于文字编辑态时接管；多选状态不执行单节点剪贴板操作。
- `Ctrl/Cmd+Z` 只在脑图画布获得焦点且活动节点不处于文字编辑态时调用脑图 History；多选状态也允许撤销上一条完整历史命令。
- 节点文字编辑态、Markdown 视图及 Obsidian 其他输入控件继续使用原生 `Ctrl/Cmd+C/X/V/Z`。
- 旧 `Alt+Shift+C/X/V/Z` 只移除默认热键，`Copy Node`、`Cut Node`、`Paste Node`、`Undo` 命令继续保留，用户仍可在 Obsidian 热键设置中自定义。
- 剪切必须等待系统剪贴板写入成功后才删除节点，避免写入失败时丢失节点；复制、剪切、粘贴命令和画布快捷键复用同一个控制器入口。

- 若实施必须改变剪贴板数据格式或支持多根粘贴，则停止并重新规划，不在本计划内增加兼容双轨。

## 验收标准

- 两个以上非根节点被选中时，按 `Backspace` 或 `Delete` 删除所有选择根及子树，父子同时选中不重复处理。
- 同父和跨父批量删除保持剩余节点顺序；一次撤销完整恢复所有原位置，一次重做再次删除。
- 删除结束后多选描边和 `aria-selected` 清理，活动节点按原主选节点的相邻兄弟/父节点规则回退。
- 脑图获得焦点且只有一个非编辑活动节点时，Windows/Linux 使用 `Ctrl+C/X/V`，macOS 使用 `Cmd+C/X/V` 完成现有节点复制、剪切和粘贴。
- 剪切只有在剪贴板写入成功后才删除节点；复制内容可以重复粘贴，粘贴仍作为当前活动节点的子节点并进入一次撤销历史。
- 多选状态不会只操作主选节点；节点文字编辑态、Markdown 编辑器和其他输入控件的原生文字剪贴板不受影响。
- 脑图非编辑态使用 `Ctrl/Cmd+Z` 一次撤销上一条完整 History 命令；节点文字编辑态及其他 Obsidian 视图保留原生撤销。
- `Alt+Shift+C/X/V/Z` 不再是默认绑定，四个命令仍存在于命令面板并可自定义热键。
- `npm run build` 通过，测试 Vault 回归无阻断问题，生成 `main.js` 与部署文件一致。

## 风险与控制

- 批量删除跨多个父节点时索引会在删除过程中变化：命令先记录原位置，同父删除按降序，撤销按父节点和原索引稳定恢复。
- 父节点与后代同时选中会导致重复处理：所有批量结构操作统一使用选择根过滤。
- 全局 Obsidian 热键会抢占文字剪贴板：标准快捷键只在脑图自己的 `keydown` 焦点链路中处理，不注册为全局默认热键。
- 异步剪贴板写入失败可能造成剪切数据丢失：只有 `writeText()` 成功后才执行删除命令。
- 异步剪贴板等待期间可能切换视图或节点：操作捕获发起时的活动 leaf 和节点，完成后必须同时复核两者仍为当前目标，避免后台删除或粘贴。
- 命令面板与画布快捷键形成双入口：两者必须委托给同一 `NodeClipboardController`，不保留两套行为实现。
