# 链接标题显示设置计划

## 目标

为脑图节点链接增加用户可配置的标题显示开关。默认保持现有清爽模式，只显示链接图标；开启后在链接图标右侧显示轻量标题文本，并保证节点宽度、分支线和子节点布局同步刷新。

## 范围

- 范围内：
  - 新增设置项 `showLinkTitle`，默认关闭。
  - 在设置页增加 `Show link title` Toggle。
  - 阅读态和编辑态链接图标均遵循该设置。
  - 开启后显示“链接图标 + 小号标题文本”，标题来自现有链接标题，标题为空时回退目标。
  - 多链接按现有链接顺序依次显示。
  - 设置切换后刷新当前打开的脑图布局。
  - 更新中英文翻译、能力文档、设计规则和变更日志。
- 范围外：
  - 右下角 minimap / navigator。
  - 链接标题样式自定义，如颜色、字号、最大宽度。
  - 将链接标题写入节点正文或恢复原始 Markdown 链接文本。
  - 改变链接点击、右键编辑、删除和跳转行为。
  - 发布版本号、Git 推送或插件市场发布。

## 假设与决策

- 默认关闭，避免改变已有脑图布局。
- 标题显示是链接附件视图层能力，不改变节点 Markdown 源文本。
- 标题显示位置为链接图标右侧，不插入节点正文内容。
- 链接标题使用轻量 muted 文本，避免干扰节点正文。
- 多链接仍使用现有链接顺序和当前链接数据源。
- 设置项属于全局插件设置，不做单文件或单节点配置。
- ADR 门禁：不需要；这是局部 UI 设置和渲染行为调整。

## 验收标准

- 默认安装或旧配置升级后，链接标题不显示，现有脑图布局不变。
- 开启设置后，链接图标右侧显示标题文本。
- 关闭设置后，标题文本消失，只保留链接图标和 hover tooltip。
- 标题显示不改变节点 Markdown 源文本。
- 多链接显示顺序稳定，点击和右键目标仍对应正确链接。
- 开关切换后节点宽度、分支线和子节点位置同步刷新。
- 深色和浅色主题下标题文本可读。
- 生产构建通过，并在本地私人插件目录完成运行验证。

## 风险

- 长标题可能导致节点过宽；第一版通过 CSS 最大宽度和截断控制。
- 链接层当前使用绝对定位，标题显示会影响占位宽度；必须同步更新链接层宽度计算。
- 编辑态和阅读态共用相同链接层渲染语义，避免出现两套显示规则。
