# 节点框选与整组迁移

## 目标

在桌面端脑图视图中实现 MindMaster 式节点框选：用户按住 `Ctrl`/`Meta` 从空白画布拖出临时选择框，保留命中节点的多选状态，并可将所选节点作为一个原子操作整组迁移和撤销。

## 范围

- `Ctrl`/`Meta` + 空白拖动，以与选择框相交的当前可见非根节点替换多选集合。
- `Ctrl`/`Meta` + 单击节点，追加或取消单个非根节点。
- 松开鼠标后移除临时框，保留节点多选视觉；空白单击或 `Escape` 清空。
- 普通空白左键手势只有在位移未超过点击容错阈值时才清空选择；实际拖动画布时保留多选状态。
- 框选期间滚轮改为上下滚动画布，选择框锚点固定在原始画布位置，并累计选择滚动经过的范围。
- 普通拖动任一已选节点，按现有兄弟/子节点落点语义迁移整组。
- 整组迁移保持稳定顺序，并以一条历史命令完成撤销和重做。
- 更新选择交互样式、设计系统规则、能力文档和变更日志。

## 非目标

- 不实现批量复制、剪切、删除、编辑、折叠或格式化。
- 不实现触摸端、移动端或套索自由曲线选择。
- 不重构整个 `mindmap.ts`、`INode.ts` 或现有单节点命令体系。
- 不改变节点 Markdown 数据结构、命令名称或插件公开入口。

## 假设与已确认决策

- 用户调用 `dev-branch` 视为确认上一轮推荐的“安全型首版语义”。
- 框选采用屏幕坐标矩形相交判断，以兼容画布缩放和滚动；连线、折叠按钮和隐藏节点不参与命中。
- 用户确认框选手势优先于缩放：框选期间 `Ctrl`/`Meta` + 滚轮只滚动画布，不改变缩放；框选结束后恢复原有缩放行为。
- 用户确认采用累计式跨屏框选：滚动时选择框起点跟随原始画布锚点移动，已进入选区但滚出视口的节点继续保持选中，反向滚动缩小几何范围时再按实时相交结果更新。
- 用户确认区分空白单击与画布拖动：左键位移不超过 4px 时按静止单击清空选择，超过阈值时判定为画布平移并抑制拖动结束产生的那一次 `click`，保留多选状态。
- `MindMap.selectNode` 继续表示唯一活动节点，兼容现有命令和键盘入口；多选集合由独立控制器持有，不把 `selectNode` 改成数组。
- 框选完成后，若原活动节点仍在集合中则保留；否则选择树遍历顺序中的第一个命中节点作为活动节点。
- 多选数大于一时暂停单节点新增、编辑和删除键盘行为，只保留 `Escape` 清空多选，避免对单个活动节点产生意外修改。
- 父节点与后代可以同时显示为已选；执行迁移时仅提交没有已选祖先的顶层选择根，后代随父节点移动。
- 根节点不进入多选集合；目标节点位于任一迁移根子树中时拒绝整组迁移。
- 整组迁移后保持选择状态。多选组不支持 `Ctrl`/`Meta` 拖动复制；带修饰键开始节点拖动时不执行整组复制。
- 不新增兼容旧多选实现的双轨逻辑；代码中历史注释不作为行为来源。

## 风险与控制

- 坐标系错误会导致缩放后误选：选择框和节点统一使用 `getBoundingClientRect()` 屏幕坐标。
- 框选滚动可能让屏幕坐标锚点失真：按容器实际 `scrollTop` 变化量反向移动选择框起点，并用最后指针位置重新计算命中，不假设请求滚动量一定生效。
- 单选与多选状态可能漂移：`selectNode` 保持现有活动节点职责，多选集合只由选择控制器修改。
- 多节点依次移动可能改变插入索引：批量命令先保存所有原父节点和原索引，再整体移除并按稳定顺序插入。
- 目标可能位于迁移子树内：执行命令前对每个迁移根做祖先守卫，任何一个非法则整组拒绝。
- HTML5 拖动与框选/平移可能竞争：只有从空白处且带 `Ctrl`/`Meta` 的左键按下进入框选；已处理手势必须阻止后续平移。
- 浏览器会在画布拖动结束后继续派发 `click`：选择控制器只在普通空白左键手势超过 4px 后，于 `mouseup` 设置短时单次点击抑制，避免长期或跨手势残留。
- 自动化测试缺失：以生产构建和隔离测试 Vault 的交互矩阵作为主要门禁，并在代码中保持几何与树归一化逻辑可独立检查。

## 验收标准

- 框选视觉和节点描边效果与参考图的交互层次一致，且使用 Obsidian 语义颜色适配深浅主题。
- 框选、单选、平移、缩放、编辑和单节点拖放之间没有手势冲突。
- 多选后普通空白拖动画布保持选择，只有位移未超过 4px 的空白左键单击或 `Escape` 才清空选择。
- 框选期间滚轮可以连续上下滚动画布而不改变缩放，选区覆盖原始锚点到当前指针之间的跨屏范围；松开鼠标后 `Ctrl`/`Meta` + 滚轮恢复缩放。
- 多选组迁移符合现有落点语义，保持顺序，不产生重复移动或循环父子关系。
- 一次撤销完整恢复整组迁移前状态，一次重做完整恢复迁移后状态。
- 多选期间不会意外触发单节点新增、编辑、删除或部分复制。
- `npm run build` 通过，测试 Vault 回归结果已记录，独立审查无阻塞问题。
